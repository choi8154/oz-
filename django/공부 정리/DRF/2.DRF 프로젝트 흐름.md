# DRF 프로젝트 흐름

<details>
<summary>MTV란?</summary>

Model Template Views 의 약자로 
- Model : DB를 정의하는 ORM(객체 관계 매핑 기술)이 있는 모델 클래스가 들어가는 계층
- Template : html형태의 응답을 해주는 사용자에게 보여지는 화면 계층
- View : 중간에서 모든 걸 조율하는 계층

</details>

<details>
<summary>MVC란?</summary>

Model View Controler 의 약자로 
- Model : DB를 정의하는 ORM(객체 관계 매핑 기술)이 있는 모델 클래스가 들어가는 계층
- View : html형태의 응답을 해주는 사용자에게 보여지는 화면 계층
- Controler : 중간에서 모든 걸 조율하는 계층

</details>

## MTV와의 차이점

### 1) Django의 MTV 구조와 DRF의 차이
- MTV는 모델 템플릿 뷰 순서로 작성하며 풀스택의 영역임. template을 따로 작성하여 연결함.
- MTV는 폼을 만들어서 템플릿에 등록하고 검증도 템플릿의 도움을 받아서 이루어짐.
- DRF는 그런 MTV에 API 개발용 레이어를 확장한 구조. MTV + SVP의 구조로
- 사실상 template이 없는 구조임. 그냥 json으로 응답함.
- 진짜 쉬멕 말하면 MTV는 html로 응답하고 DRF는 json으로 응답함.
